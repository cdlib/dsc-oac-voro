In order to make running the LHDRP 7train process that we run every year
a little easier, maintenance-wise, Brian figured it would be better to
maintain the associations in an easily modified file.  A BerkeleyDB file
seemed ideal for this.  Brian figured that the way to make the information
accessible to XSLT was through a Java static method.  Oracle is providing
a free implementation of BerkeleyDB in Java.  On page
"http://www.oracle.com/technology/products/berkeley-db/index.html", I clicked
"Berkeley DB Java Edition", which took me to page
"http://www.oracle.com/technology/products/berkeley-db/je/index.html".  I
clicked the "Download" link (near the top on the right hand side of the page),
which took me to page:
"http://www.oracle.com/technology/software/products/berkeley-db/je/index.html".
Under "Latest Production Release", under "3.2.76", I clicked the
"Berkeley DB Java Edition 3.2.76.tar.gz (4.7M)" link (which downloaded the
file, and brought up pop-up:
"http://www.oracle.com/technology/software/products/berkeley-db/htdocs/popup/je/3.2.76/je-targz.html").
